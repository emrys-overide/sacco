# Sowetamu SACCO user guide and technical help

The in-application copy of this guide is available at `/documentation`. It is
safe to share with SACCO members and officers; it deliberately contains no
database credentials, recovery secrets, bank credentials, or other internal
security configuration.

## Getting started

- **Members:** create an account only with the name, phone number, and email
  already recorded on your active SACCO member record. Then sign in using the
  registered phone number or email and your password.
- **Officers:** the Chairman creates the officer account and assigns the role.
  Officers must use individual accounts and must never share passwords.
- **Loans:** members apply in the Loans section, follow each review stage, and
  can see any rejection reason before submitting a new application.

## Forgotten password

Use **Request password reset** on the sign-in screen, then contact the Chairman
or SACCO Administrator directly. The system sends them an in-app notification
for a valid request. After they verify the member's identity, they supply a
temporary password. The member must use that password only to sign in and
immediately create a new private password.

## Notifications

The bell delivers workflow notifications while the application is online. An
item remains in the account when a user is offline and is shown after they
reconnect and sign in.

## Safe use

- Never share a password, temporary password, authenticator code, or bank
  information.
- Log out from shared devices.
- Report unexpected account access or a wrong financial record promptly.
- When reporting a technical issue, include the user role, page, time, and a
  screenshot where possible. Do not include secrets in the report.
